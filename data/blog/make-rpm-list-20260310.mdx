## はじめに

サーバーに導入されているソフトウェアの一覧を、最新の状態で管理したい場面があります。たとえば次のような情報を揃えておきたいケースです。

- ソフトウェア名
- バージョン番号
- 当該ソフトウェアが導入されている機器を特定するための情報
- ベンダサポート期限

本記事では、Amazon Linux 2023環境でこの台帳をコマンド2つ + PowerQueryで作成する方法を紹介します。

## 方式選定 ── なぜこの方法にしたか

### 本来のベストプラクティス

AWSでソフトウェアインベントリを管理するなら、まず検討すべきは以下のマネージドサービスです。

| サービス                          | できること                      |
| ----------------------------- | -------------------------- |
| AWS Systems Manager Inventory | EC2のソフトウェア・OS情報を自動収集し、一元管理 |
| Amazon Inspector              | パッケージの脆弱性スキャン + インベントリ取得   |

どちらもエージェントベースで自動収集でき、運用負荷は低いです。**使える環境なら、これらを使うのが正解です。**

### マネージドサービスが使えないケースを想定する

一方で、これらのマネージドサービスが使えない環境も現実には存在します。たとえば次のような制約です。

- IAM の権限が絞られていて、SSM や Inspector の API を呼べない
- SSM エージェントがそもそも導入されていない
- マネージドサービスの導入自体に承認プロセスが必要で、すぐには使えない

こうした制約は珍しくありません。本記事は、そうした「マネージドサービスに頼れない環境」でも、OS標準のコマンドだけで台帳を作る方法を扱います。

### 代替手段の選定

制約を踏まえると、方針は次のようになります。

- **取得方法**：sshでサーバーに入り、`rpm -qa` と `dnf supportinfo` で必要な情報をCSV出力する
- **加工方法**：PowerQueryで2つのCSVをジョインしてExcel化する

PowerQueryを選ぶ理由は、**最終成果物がExcelで求められる場合があること**と、**非エンジニアの運用担当にも引き継ぎやすいこと**です。Pythonのpandasでも同じことはできますが、引き継ぎ先がExcelを日常的に使っている環境では、Excel完結のほうが運用が続きやすいです。

## 環境

- OS：Amazon Linux 2023
- ローカル端末：Windows（Excel + PowerQuery）
- 接続方法：ssh

## 手順

### 1. サーバー情報の取得

まず、台帳の「機器を特定するための情報」としてサーバーの基本情報を取得します。

```bash
# カーネルバージョン
uname -r

# OS名・バージョン
cat /etc/os-release

# ホスト名
hostname

# ハードウェア概要
lshw -short

# ネットワークインターフェースのIP
ip addr show

# ルーティング設定
ip route
```

これらの出力は台帳のヘッダー情報として使います。

### 2. rpmパッケージ一覧の取得

以下のコマンドで、インストール済みパッケージの一覧をCSV形式で出力します。

```bash
rpm -qa --qf '%{installtime} %{name} %{version} %{release} %{vendor}\n' \
  | awk '{
      cmd = "date -d @" $1 " +\"%Y年%m月%d日 %H:%M:%S\""
      cmd | getline d
      close(cmd)
      print d "," $2 "," $3 "," $4 "," $5
    }'
```

**このコマンドが何をしているか：**

- `rpm -qa --qf` でインストール日時（UNIXタイムスタンプ）、パッケージ名、バージョン、リリース、ベンダーを出力
- `awk` 内で `date` コマンドを呼び、UNIXタイムスタンプを日本語の日付形式に変換
- カンマ区切りで出力し、CSVとして扱えるようにする

出力結果をローカルに `rpm.csv` として保存します。

### 3. サポート期限情報の取得

AL2023には `dnf supportinfo` というコマンドがあり、各パッケージのベンダサポート期限を取得できます。

```bash
dnf supportinfo --show installed | awk '{print $1","$2","$5}'
```

出力結果をローカルに `dnf.csv` として保存します。

> **補足**：`dnf supportinfo` はAmazon Linux 2023固有の機能です。他のディストリビューションでは使えないため、AL2023を使っている場合にこの方法が活きます。

### 4. PowerQueryでジョインしてExcel化

ここからはローカルのExcelで作業します。

#### 4-1. PowerQueryエディタを開く

Excelを起動し、「データ」タブ →「データの取得」→「Power Queryエディターの起動」を選択します。

#### 4-2. CSVの読み込み

「新しいソース」→「ファイル」→「CSV」で、`rpm.csv` と `dnf.csv` をそれぞれ読み込みます。

#### 4-3. テーブルのジョイン

`rpm.csv` のクエリを選択した状態で、「ホーム」→「クエリのマージ」から `dnf.csv` をジョインします。

- **ジョインキー**：パッケージ名の列を指定
- **結合の種類**：左外部結合（rpm側を全件残す）

#### 4-4. 列の整理と出力

不要な列を削除し、列名を「ソフトウェア名」「バージョン」「リリース」「ベンダー」「インストール日」「サポート期限」などに変更します。

「閉じて読み込む」でExcelシートに出力すれば完成です。

## ハマりどころ

### dnf supportinfoの出力形式

`dnf supportinfo` の出力は固定幅フォーマットのため、`awk` でフィールドを切り出す際にカラム位置がずれることがあります。実際の出力を確認してから `awk` のフィールド番号を調整してください。

### パッケージ名の表記揺れ

`rpm -qa` と `dnf supportinfo` でパッケージ名の表記が微妙に異なるケースがあります（アーキテクチャ `.x86_64` の有無など）。PowerQueryでジョインする前に、どちらかに合わせて正規化しておくとマージ漏れが減ります。

### CSVのエンコーディング

PowerQueryでCSVを読み込む際、デフォルトのエンコーディングがUTF-8でない場合があります。文字化けしたら読み込み時に「元のファイル」→「65001: Unicode (UTF-8)」を指定してください。

## まとめ

SSM InventoryやInspectorが使える環境なら、そちらを使うのが正解です。ただし、それらのマネージドサービスが使えない環境では、`rpm` + `dnf supportinfo` + PowerQueryという組み合わせでソフトウェア台帳を作成できます。

ポイントは以下の3点です。

- `dnf supportinfo` でAL2023のサポート期限を取得できる（AL2023固有の機能）
- PowerQueryを使うことで、非エンジニアにも引き継ぎ可能な運用フローになる
- マネージドサービスが使えない場合の代替手段として、OS標準コマンドで十分に対応できる
